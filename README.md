# 🐧 Administración de Sistemas Linux – Prácticas de Red Hat Enterprise Linux

## 📖 Descripción

Este repositorio contiene el desarrollo de tres prácticas orientadas a la administración de sistemas Linux utilizando Red Hat Enterprise Linux (RHEL). Las actividades realizadas abarcan la gestión de paquetes y repositorios, la automatización de tareas administrativas y la administración de almacenamiento mediante discos y sistemas de archivos.

El objetivo principal es fortalecer las competencias necesarias para desempeñarse como administrador de sistemas y adquirir experiencia práctica en entornos empresariales basados en Linux.

---

# 🎯 Objetivos

* Administrar paquetes y repositorios del sistema operativo.
* Automatizar tareas recurrentes mediante herramientas de programación.
* Gestionar dispositivos de almacenamiento y sistemas de archivos.
* Aplicar buenas prácticas de administración en servidores Linux.
* Desarrollar habilidades fundamentales para la administración de infraestructuras basadas en Red Hat.

---

# 🛠️ Práctica 1 – Gestión de Paquetes y Repositorios

Durante esta práctica se realizó la actualización completa del sistema operativo para garantizar que todos los paquetes estuvieran en sus versiones más recientes.

Posteriormente, se identificaron y verificaron los repositorios configurados en el servidor para conocer las fuentes de software disponibles. Se investigó la disponibilidad de la herramienta **Bashtop Resource Monitor**, utilizada para monitorear recursos del sistema en tiempo real.

Una vez instalada y verificado su correcto funcionamiento, se procedió a su desinstalación completa, eliminando tanto el software como los archivos de configuración asociados. Finalmente, se realizó la limpieza de dependencias que ya no eran necesarias para mantener el sistema optimizado.

### Competencias desarrolladas

* Gestión de repositorios.
* Instalación y eliminación de software.
* Administración de dependencias.
* Mantenimiento y actualización de sistemas Linux.

---

# ⏰ Práctica 2 – Automatización de Tareas

En esta práctica se configuraron tareas programadas para automatizar procesos administrativos comunes dentro de un servidor Linux.

Se estableció una tarea automática para mantener el sistema actualizado diariamente durante horas de baja actividad. Además, se programó un reinicio semanal del servidor con fines de mantenimiento preventivo.

Como complemento, se utilizó la herramienta de programación de tareas diferidas para ejecutar una acción específica después de un intervalo de tiempo determinado, verificando posteriormente que la operación se realizara correctamente.

### Competencias desarrolladas

* Automatización de tareas.
* Administración de procesos programados.
* Mantenimiento preventivo de servidores.
* Gestión de tareas diferidas.

---

# 💽 Práctica 3 – Administración de Discos y Sistemas de Archivos

La tercera práctica estuvo enfocada en la gestión de almacenamiento.

Se agregó un nuevo disco virtual al sistema y se preparó para su utilización mediante la creación de una partición y la configuración de un sistema de archivos compatible con Linux.

Posteriormente, se realizó el montaje del disco en un directorio específico del sistema para habilitar su acceso. Dentro de la nueva unidad se creó un archivo de prueba con el propósito de verificar el correcto funcionamiento del almacenamiento.

Finalmente, se desmontó y volvió a montar la unidad en una ubicación diferente, comprobando que los datos permanecieran intactos después del proceso.

### Competencias desarrolladas

* Gestión de discos y particiones.
* Administración de sistemas de archivos.
* Montaje y desmontaje de unidades.
* Verificación de persistencia de datos.
